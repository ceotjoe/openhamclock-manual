🎛️ Rig Bridge

Feature

Click any DX spot, POTA activator, or satellite frequency and your radio tunes there instantly. Rig Bridge connects OpenHamClock to your transceiver over USB serial, CI-V, or network.

What Rig Bridge Does

Rig Bridge is OpenHamClock's rig control layer. Once connected, every frequency shown in the dashboard becomes a clickable link to your radio. Click a DX spot on 14.025 MHz CW and your transceiver jumps there in CW mode — without you touching the VFO. Click a POTA activator on 7.225 MHz SSB and the radio switches to 40 m LSB. Click a WSJT-X spot and the radio moves to the FT8 dial frequency in USB.

The Rig Control panel also shows your radio's current frequency, mode, and VFO in real time — a permanent readout visible from across the shack, even on a wall-mounted display.

ℹ️
Not sure which protocol your radio uses?

Check your radio's manual under "PC Control" or "CAT System". Yaesu radios use CAT (often via a USB cable that enumerates as a COM/tty port). Icom radios use CI-V (via USB or a dedicated CI-V interface). Kenwood uses a simple ASCII protocol. Most modern radios present as a USB serial device — no extra interface needed.

Setup: Choosing Your Connection Method

There are two ways to connect your radio to OpenHamClock:

Option A — rig-listener (recommended)

A small standalone executable that runs on any Windows, macOS, or Linux computer connected to your radio. It handles the serial connection and exposes a simple HTTP API that OpenHamClock calls over the network.

Best for: any setup. No Node.js required on the radio PC. Works when OpenHamClock is on a different machine (Pi, cloud, etc.) than the radio.

Option B — direct from server

OpenHamClock's Node.js server connects to the radio's serial port directly. Simpler, but requires OpenHamClock to be running on the same machine as the radio.

Best for: single-machine setups where OpenHamClock and your radio are both connected to the same computer.

Setting Up rig-listener (Option A)

  1. Download rig-listener

    Go to the OpenHamClock releases page and download the latest rig-listener binary for your OS: rig-listener-win.exe, rig-listener-macos, or rig-listener-linux.

  2. Connect your radio via USB and find the serial port

    On Windows: open Device Manager → Ports (COM & LPT). Note the COM number (e.g. COM4).
    On macOS: run ls /dev/tty.usbserial* or ls /dev/cu.*.
    On Linux: run ls /dev/ttyUSB* or ls /dev/ttyACM*.

  3. Run rig-listener with your radio's parameters
    bash — Icom IC-7300 example 
    $ ./rig-listener --port /dev/ttyUSB0 --baud 9600 --rig icom-ci-v --ci-v-address 0x94
    bash — Yaesu FT-991A example 
    $ ./rig-listener --port COM4 --baud 38400 --rig yaesu-cat

    rig-listener starts an HTTP server on port 3005 by default.

  4. Point OpenHamClock at rig-listener

    In the OpenHamClock Settings panel, open Rig Control and set the URL to:

    Settings → Rig Control → URL 
    http://localhost:3005      # if rig-listener is on the same machine
http://192.168.1.50:3005   # if rig-listener is on a different machine

    Click Test Connection. The Rig Control panel should show your radio's current frequency.

Direct Server Setup (Option B)

Add these variables to your .env and restart the server:

.env 
RIG_ENABLED=true
RIG_TYPE=icom-ci-v           # see table below for options
RIG_SERIAL_PORT=/dev/ttyUSB0  # COM4 on Windows
RIG_BAUD_RATE=9600
RIG_CI_V_ADDRESS=0x94         # Icom only — check your radio's manual

Rotator Control

OpenHamClock also supports antenna rotator control. When a DX spot or activator is selected, the bearing to that station is computed from your grid square and displayed in the Rotator panel. If a rotator controller is connected, clicking Rotate sends the azimuth command automatically.

A bearing line from your QTH to the DX station is drawn on the world map, giving you a visual confirmation that your beam is pointed in the right direction.

💡
No rotator? Still useful.

Even without a connected rotator, the bearing readout and map line are valuable. If you have a fixed yagi or manually-rotated beam, the azimuth shown in OpenHamClock tells you exactly where to turn it before you call. No separate bearing calculator needed.

The Rig Control Panel

Once connected, the Rig Control panel shows a live readout of your radio's state:

Frequency Current VFO frequency, large and readable from across the shack.
Mode Current operating mode: CW, USB, LSB, FT8, RTTY, AM, FM.
VFO VFO A or VFO B indicator.
PTT state Transmitting / receiving indicator. Updates in real time.
Band Derived from frequency — shown as band name (e.g. "20 m").
⚠️
Serial port permissions on Linux

On Linux, your user account may not have permission to open /dev/ttyUSB0 by default. Add yourself to the dialout group and log out/in:

sudo usermod -aG dialout $USER

On Raspberry Pi OS this is not needed — the default pi user already has dialout access.
